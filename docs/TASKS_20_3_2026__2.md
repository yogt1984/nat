# Implementation Plan: Real-Time Dashboard via WebSocket

**Date**: 2026-03-20
**Objective**: Expose ingestor logs and market state via WebSocket for remote monitoring.
**Target**: `make run_and_serve` provides a URI accessible from anywhere.

---

## Overview

### Goal
Run the ingestor at home, monitor its state and logs from any device via web browser.

### Architecture
```
┌──────────────────────────────────────────────────────────┐
│                      ING Ingestor                        │
│  ┌─────────────┐    ┌─────────────────┐    ┌──────────┐  │
│  │ MarketState │───▶│ Dashboard Server │◀──│ Tracing  │  │
│  │ + Features  │    │  (port 8080)     │   │  Logs    │  │
│  └─────────────┘    └────────┬─────────┘   └──────────┘  │
└──────────────────────────────┼───────────────────────────┘
                               │
                    ┌──────────▼──────────┐
                    │  Cloudflare Tunnel  │  (or ngrok)
                    │  https://ing.your   │
                    │  domain.com         │
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │      Browser        │
                    │  (anywhere)         │
                    └─────────────────────┘
```

### Endpoints
| Endpoint | Type | Description |
|----------|------|-------------|
| `/` | HTTP | Static HTML dashboard |
| `/ws/logs` | WebSocket | Real-time log stream |
| `/ws/state` | WebSocket | Market state updates (1/sec) |
| `/api/symbols` | HTTP | List of tracked symbols |
| `/health` | HTTP | Health check for monitoring |

### Task 1.4: Implement WebSocket Handlers
**Priority**: P0
**Effort**: 2h

**Implementation** (`dashboard/handlers.rs`):
```rust
pub async fn logs_handler(
    ws: WebSocketUpgrade,
    State(state): State<Arc<DashboardState>>,
) -> impl IntoResponse {
    ws.on_upgrade(|socket| handle_logs_socket(socket, state))
}

async fn handle_logs_socket(mut socket: WebSocket, state: Arc<DashboardState>) {
    let mut rx = state.log_tx.subscribe();

    // Send last N logs as backfill
    for log in state.log_buffer.read().iter() {
        let _ = socket.send(Message::Text(serde_json::to_string(log).unwrap())).await;
    }

    // Stream new logs
    while let Ok(log) = rx.recv().await {
        if socket.send(Message::Text(serde_json::to_string(&log).unwrap())).await.is_err() {
            break;
        }
    }
}
```

**Acceptance Criteria**:
- [ ] New clients receive last 100 log entries
- [ ] Handles client disconnect gracefully
- [ ] No memory leak from abandoned connections

### Task 1.5: Implement HTTP Server
**Priority**: P0
**Effort**: 1h

**Implementation** (`dashboard/server.rs`):
```rust
pub async fn run_dashboard_server(
    addr: SocketAddr,
    state: Arc<DashboardState>,
) -> Result<()> {
    let app = Router::new()
        .route("/", get(serve_index))
        .route("/ws/logs", get(logs_handler))
        .route("/ws/state", get(state_handler))
        .route("/api/symbols", get(symbols_handler))
        .route("/health", get(|| async { "ok" }))
        .nest_service("/static", ServeDir::new("static"))
        .layer(CorsLayer::permissive())
        .with_state(state);

    let listener = TcpListener::bind(addr).await?;
    info!(%addr, "Dashboard server started");
    axum::serve(listener, app).await?;
    Ok(())
}
```

**Acceptance Criteria**:
- [ ] Server starts on configured port
- [ ] CORS enabled for local development
- [ ] Health endpoint returns 200

class Dashboard {
    constructor() {
        this.connectLogs();
        this.connectState();

        this.logsWs = new WebSocket(`ws://${location.host}/ws/logs`);

## Phase 4: Remote Access

### Task 4.1: Setup Cloudflare Tunnel (Recommended)
**Priority**: P1
**Effort**: 1h

**Steps**:
1. Install cloudflared: `brew install cloudflared` / `apt install cloudflared`
2. Login: `cloudflared login`
3. Create tunnel: `cloudflared tunnel create ing`
4. Configure DNS: Point subdomain to tunnel
5. Run: `cloudflared tunnel run ing`

**Benefits over ngrok**:
- Free for personal use
- Stable URLs (no random subdomains)
- Better security (Cloudflare WAF)

**Alternative**: Use Tailscale VPN for private access

**Acceptance Criteria**:
- [ ] Dashboard accessible via HTTPS URL
- [ ] Works from mobile outside home network
