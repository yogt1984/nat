### Task 3: Implement spectral derivatives

**File:** `scripts/cluster_pipeline/derivatives.py`

**Problem:** The spec defines spectral features (low-freq power, high-freq power, spectral ratio, dominant period) that capture periodicity and frequency-domain regime changes. These are entirely missing. The `include_spectral` flag is a no-op.

**Why it matters:** Spectral features detect oscillating regimes (mean-reverting vs trending) that temporal derivatives alone miss. A market switching between "oscillating in a range" and "trending" has identical velocity/acceleration distributions but completely different spectral profiles.

**Implementation:**
```python
def spectral_derivatives(bars, columns, windows=[30]):
    for col in columns:
        for w in windows:
            X_k = np.fft.rfft(window_data)
            power = np.abs(X_k)**2
            spec_low = power[:w//5].sum()
            spec_high = power[4*w//5:].sum()
            spec_ratio = spec_low / (spec_high + 1e-10)
            dominant_period = w / np.argmax(power[1:]) if max(power[1:]) > 2*mean(power) else NaN
```

**Also:** Add spectral kill criterion — if spectral features don't increase explained variance in PCA by > 5%, drop them (avoids noise injection).

**Effort:** 3h (implement + test + integrate into `generate_derivatives`)

### Task 4: Break detection should use PCA-reduced space

**File:** `scripts/cluster_pipeline/hierarchy.py` → `_detect_breaks_safe()`

**Problem:** Running PELT on 88+ raw columns dilutes statistical power. The BIC penalty scales with n_features, so high-dimensional input makes PELT overly conservative (fewer breaks detected than truly exist). The spec says "first 5 PCA components" — this concentrates variance and gives PELT a cleaner signal.

**Fix:**
```python
def _detect_breaks_safe(bars, numeric_cols):
    X = bars[numeric_cols].fillna(0).values
    # Quick PCA to 5 components
    from sklearn.decomposition import PCA
    n_comp = min(5, X.shape[1], X.shape[0])
    X_pca = PCA(n_components=n_comp).fit_transform(X)
    # PELT on reduced space
    pen = np.log(len(X_pca)) * n_comp  # much smaller penalty
    algo = rpt.Pelt(model="rbf", min_size=50).fit(X_pca)
    ...
```

**Risk:** May detect MORE breaks (smaller penalty), causing shorter segments. Need to verify on real data that this doesn't over-fragment.

**Effort:** 1h
