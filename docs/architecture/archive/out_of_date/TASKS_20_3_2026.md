# Implementation Plan: Regime Detection Feature Extraction

**Date**: 2026-03-20
**Objective**: Extend feature extraction pipeline with regime detection features (absorption, volume-price divergence, churn rate, range position) and validate cluster structure in the extended feature space.

---

## Overview

### Core Hypothesis
Market states (Accumulation, Distribution, Markup, Markdown, Neutral) manifest as clusters in a 5-dimensional feature space defined by:
1. **Absorption Ratio** - Volume per unit price change
2. **Volume-Price Divergence** - Deviation from expected price impact
3. **Whale Flow (Normalized)** - Net whale position change
4. **Churn Rate** - Two-sided trading activity
5. **Range Position** - Location within recent price range

### Success Criteria
- [ ] Regime features integrated into ingestion pipeline
- [ ] Features computed at multiple timeframes (1h, 4h, 24h for regime; 4h, 24h, 1w for range)
- [ ] Cluster analysis reveals 4-6 distinct states
- [ ] Clusters align with theoretical market phases
- [ ] Forward returns differ significantly between clusters

## Time Scale Design

### Rationale
Regime features detect accumulation/distribution phases that unfold over **hours to days**, not minutes. This is fundamentally different from microstructure features.

| Feature Type | Timescale | Resolution | Rationale |
|--------------|-----------|------------|-----------|
| **Microstructure** (existing) | Seconds to minutes | 100ms ticks | Order flow, toxicity, immediate imbalance |
| **Regime** (new) | Hours to days | 1-minute bars | Accumulation, distribution, market phases |

### Time Windows

| Feature | Short | Medium | Long | Very Long |
|---------|-------|--------|------|-----------|
| **Absorption** | 1h (60) | 4h (240) | 24h (1440) | - |
| **Volume-Price Divergence** | 1h (60) | 4h (240) | 24h (1440) | - |
| **Churn Rate** | 1h (60) | 4h (240) | 24h (1440) | - |
| **Range Position** | 4h (240) | 24h (1440) | 1 week (10080) | - |

*(Values in parentheses are minutes)*

### Market Structure Alignment

| Cycle | Duration | Relevance |
|-------|----------|-----------|
| Funding rate | 8 hours | Whales adjust positions around funding |
| Trading session | 8-12 hours | Session-based accumulation patterns |
| Daily | 24 hours | Full daily regime cycle |
| Weekly rebalance | 168 hours | Institutional position adjustment |

### Buffer Requirements

At 1-minute resolution (downsampled from 100ms):

| Window | Samples Needed | Memory (per symbol) |
|--------|----------------|---------------------|
| 1 hour | 60 | ~2.4 KB |
| 4 hours | 240 | ~9.6 KB |
| 24 hours | 1,440 | ~58 KB |
| 1 week | 10,080 | ~400 KB |

**Total per symbol**: ~500 KB (trivial)

### Computation Frequency

Regime features are computed at **1-minute intervals**, not every 100ms tick:
- Reduces computation by 600x
- More stable signals (less noise)
- Aligns with the natural timescale of the phenomena

### Task 1.3: Implement Volume-Price Divergence
**Priority**: P0
**Estimated Effort**: 3 hours
**Dependencies**: Task 1.1

**Description**:
Implement volume-price divergence using Kyle's lambda estimation.

**Mathematical Definition**:
```
Expected_ΔP = λ * SignedVolume
Actual_ΔP = Price[t] - Price[t-w]
Divergence = Actual_ΔP - Expected_ΔP
```

Where `λ` is estimated via rolling regression:
```
λ = Cov(ΔP, SignedVol) / Var(SignedVol)
```

**Implementation** (`regime/divergence.rs`):
```rust
pub struct DivergenceComputer {
    price_buffer: RingBuffer<f64>,
    buy_volume_buffer: RingBuffer<f64>,
    sell_volume_buffer: RingBuffer<f64>,
    kyle_lambda: f64,
    lambda_update_window: usize,
}

impl DivergenceComputer {
    pub fn update_kyle_lambda(&mut self);
    pub fn compute(&self, window: usize) -> f64;
    pub fn compute_zscore(&self, window: usize) -> f64;
    pub fn get_kyle_lambda(&self) -> f64;
}
```

**Acceptance Criteria**:
- [ ] Kyle's lambda updates correctly via rolling regression
- [ ] Divergence is negative when buying doesn't push price up (accumulation)
- [ ] Divergence is positive when selling doesn't push price down (distribution)
- [ ] Handles zero signed volume gracefully
- [ ] Lambda estimation is stable (not too noisy)

**Tests**:
```rust
#[test]
fn test_kyle_lambda_estimation() {

import numpy as np
import pandas as pd
from sklearn.preprocessing import StandardScaler
from sklearn.mixture import GaussianMixture
from sklearn.cluster import HDBSCAN
import matplotlib.pyplot as plt

def analyze_clusters(parquet_path):
    # Load data
    df = pd.read_parquet(parquet_path)

    # Select regime features
    regime_cols = [
        'regime_absorption_zscore',
        'regime_vpd_zscore',
        'whale_flow_normalized_1h',  # From whale_flow features
        'regime_churn_zscore',
        'regime_range_pos_4h',
    ]

    X = df[regime_cols].dropna()
    X_scaled = StandardScaler().fit_transform(X)

    # GMM clustering
    best_bic = np.inf
    best_gmm = None
    for n in range(3, 8):
        gmm = GaussianMixture(n_components=n, covariance_type='full')
        gmm.fit(X_scaled)
        if gmm.bic(X_scaled) < best_bic:
            best_bic = gmm.bic(X_scaled)
            best_gmm = gmm

    labels = best_gmm.predict(X_scaled)

    # Characterize clusters
    characterize_clusters(df.loc[X.index], labels, regime_cols)

    # Visualize
    visualize_clusters(X_scaled, labels)

    return best_gmm, labels
```

def validate_cluster_predictive_power(df, labels, horizons=[60, 240, 1440]):
    results = []

    for horizon in horizons:
        # Forward returns
        df['fwd_return'] = df['midprice'].pct_change(horizon).shift(-horizon)

        for cluster in np.unique(labels):
            mask = labels == cluster
            returns = df.loc[mask, 'fwd_return'].dropna()

            results.append({
                'horizon': horizon,
                'cluster': cluster,
                'mean_return': returns.mean(),
                'std_return': returns.std(),
                't_stat': returns.mean() / (returns.std() / np.sqrt(len(returns))),
                'n_samples': len(returns),
            })

    return pd.DataFrame(results)
```

def label_cluster(cluster_profile):
    """
    cluster_profile: dict with mean values of regime features
    """
    absorption = cluster_profile['absorption_zscore']
    whale_flow = cluster_profile['whale_flow_normalized']
    range_pos = cluster_profile['range_position']
    churn = cluster_profile['churn_zscore']
    fwd_return = cluster_profile.get('fwd_return_4h', 0)

    is_absorbing = absorption > 0.5
    is_high_churn = churn > 0.5

    if is_absorbing and is_high_churn:
        if whale_flow > 0.3 and range_pos < 0.4:
            return 'ACCUMULATION'
        elif whale_flow < -0.3 and range_pos > 0.6:
            return 'DISTRIBUTION'
        else:
            return 'CONSOLIDATION'
    elif fwd_return > 0.001:
        return 'MARKUP'
    elif fwd_return < -0.001:
        return 'MARKDOWN'
    else:
        return 'NEUTRAL'
```
