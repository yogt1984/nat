## Milestone 1: Cluster Detection in Feature Space

### Core Question
> Do market states form natural clusters in 183-dimensional feature space?

### Expected Answer
No — not in full 183D space. Markets are continuously distributed with smooth
transitions, not discrete states.

### Productive Approach
Cluster **subspaces**, not the full feature vector:

| Subspace | Features | Expected Clustering |
|----------|----------|-------------------|
| Entropy only | ent_permutation_8/16/32, ent_book_shape, ent_trade_size | Moderate (2 clusters via BIC on synthetic) |
| Volatility only | vol_realized_20/100, vol_ratio | Good (low-vol / high-vol well-documented) |
| Flow only | whale_net_flow, flow_aggressor, imbalance | Weak at tick level |
| Combined best 2-3 | entropy + volatility + whale flow | Best candidate for composite regime |

### Separation Measurement
- Silhouette score > 0.25 = reasonable, > 0.5 = strong
- Davies-Bouldin < 1.0 = good
- Kruskal-Wallis on forward returns by cluster: p < 0.05

## Milestone 2: Timeframe Exploration

### The Critical Gap
Tick-level (100ms) features are too noisy. Transaction costs (13 bps round-trip)
destroy any edge at 0.07% average tick return.

### Multi-Horizon Aggregation Strategy

| Timeframe | Aggregation | Expected Clustering Quality |
|-----------|-------------|---------------------------|
| Tick (100ms) | Raw | Poor — noise dominates |
| 1-min bars | mean, std, min, max per feature | Slightly better |
| 5-15 min bars | Same + entropy slope | **Best candidate for regime detection** |
| 1-hour bars | Same + cumulative whale flow | Good for volatility regimes |
| 4h-daily | Full statistical profile | Accumulation/distribution visible |

### Cluster Derivatives
Beyond static cluster labels, compute:
- **Transition rate**: rate of change between cluster assignments
- **Dwell time**: how long market stays in a cluster before transitioning
- **Transition matrix**: probability of moving from cluster A to cluster B
- **Transition momentum**: is the system accelerating toward a new cluster?

These derivatives may be more informative than the cluster labels themselves.

## Milestone 3: Clustering Algorithms & Quality Metrics

### Algorithm Catalog

#### Partition-Based
| Algorithm | Strengths | Weaknesses | Use Case |
|-----------|-----------|------------|----------|
| K-Means | Fast, simple | Assumes spherical clusters, sensitive to outliers | Baseline |
| K-Medoids (PAM) | Robust to outliers, any distance metric | Slower | When outliers are a concern |
| **GMM** | Probabilistic, elliptical clusters, soft assignments | Assumes Gaussian components | **Primary choice** — gives P(trending)=0.7, P(ranging)=0.3 |

#### Density-Based
| Algorithm | Strengths | Weaknesses | Use Case |
|-----------|-----------|------------|----------|
| DBSCAN | Finds arbitrary shapes, identifies noise | Sensitive to eps parameter | Finding "normal" vs "anomalous" market |
| HDBSCAN | Auto cluster count, varying densities | Slower, less interpretable | Natural grouping discovery |
| OPTICS | Reachability plot shows cluster structure | Complex to tune | Visual cluster structure analysis |

#### Hierarchical
| Algorithm | Strengths | Weaknesses | Use Case |
|-----------|-----------|------------|----------|
| Agglomerative (Ward) | Dendrogram reveals hierarchy | O(n^2) memory | If regimes have sub-regimes |

#### Temporal/Model-Based
| Algorithm | Strengths | Weaknesses | Use Case |
|-----------|-----------|------------|----------|
| **HMM** | Models temporal transitions, regime persistence | Must pre-specify states | **Most natural for market regimes** |
| Bayesian GMM (DPGMM) | Auto-determines cluster count | Computationally expensive | When unsure of number of regimes |
| Spectral Clustering | Finds non-convex clusters | Scalability | Nonlinear regime boundaries |

### Quality Metrics

#### Internal Metrics (No Ground Truth)
| Metric | Range | Good Value | Measures |
|--------|-------|------------|----------|
| Silhouette Score | [-1, 1] | > 0.5 strong, > 0.25 reasonable | Within vs between cluster similarity |
| Calinski-Harabasz | [0, inf) | Higher = better | Between/within cluster variance ratio |
| Davies-Bouldin | [0, inf) | < 1.0 | Average cluster similarity to nearest |
| Gap Statistic | R | Largest gap = optimal k | Dispersion vs null reference |
| BIC/AIC (GMM) | R | Minimum = optimal components | Model fit vs complexity |

#### Multimodality Tests (Per Feature or Projection)
| Test | Output | Interpretation |
|------|--------|---------------|
| Hartigan's Dip Test | p-value | p < 0.05 = multimodal |
| Silverman Bandwidth Test | Mode count | Number of natural modes |
| Bimodality Coefficient | BC value | BC > 0.555 = bimodal |
| KDE Peak Detection | Mode locations | Where modes are |
| GMM BIC Selection | Component count | Optimal Gaussian components |

#### External Validation (Against Outcomes)
| Metric | What It Tests |
|--------|--------------|
| Kruskal-Wallis by cluster | Do forward returns differ across clusters? |
| Eta-squared | How much return variance explained by cluster? |
| Self-transition rate | Do clusters persist (> random)? |
| Cluster-conditional Sharpe | Trading edge within cluster? |

#### Stability Metrics
| Metric | What It Tests |
|--------|--------------|
| Bootstrap ARI | Reproducibility under resampling |
| Temporal ARI | First-half vs second-half consistency |
| Cluster center CV | Center consistency across samples |

**Key principle:** A cluster is only useful for trading if it (a) differentiates forward
returns AND (b) is stable across time. Beautiful silhouette means nothing without (a).

## Milestone 4: Cluster Publishing Bots

### Architecture
```
NAT Ingestor (Rust, live)
    -> Feature Parquet (tick-level)
    -> Aggregation service (Python, every N minutes)
    -> Fit/apply pre-trained clustering model (GMM/HMM)
    -> Publish: { timestamp, cluster_id, cluster_probs, features_summary }
```

### Publish Targets (Staged)
1. **Stage 1:** JSON file on disk (simplest, good for development)
2. **Stage 2:** SQLite database (queryable history)
3. **Stage 3:** WebSocket channel (real-time consumers)
4. **Stage 4:** REST API (external consumers)

**Timing:** Do NOT build until Milestones 1-3 confirm clusters exist and are tradeable.
The publisher is an afternoon of work once you have a validated model.

## Milestone 5: Dimensionality Reduction for Visualization

### Methods (Ranked by Usefulness)

#### Linear
- **PCA (to 2D/3D):** Preserves global variance. Good first look. We know 90% of
  variance is in 12/19 features — PCA compresses well.

#### Nonlinear (Manifold Learning)
- **UMAP (recommended primary):** Fast, preserves global structure, reproducible.
  Better than t-SNE for comparing clusters across runs.
- **t-SNE:** Excellent local structure visualization. Warning: distances between
  clusters are meaningless, only within-cluster structure is interpretable.
  Hyperparameter-sensitive (perplexity).
- **Isomap:** Geodesic distance-preserving. Good if market state space is a
  curved manifold (likely true).

### Visualization Protocol
Run UMAP to 3D on aggregated features. Generate 4 plots, each colored by:
1. Entropy level
2. Volatility regime
3. Forward return sign
4. Cluster assignment

**If the same structure appears regardless of coloring, the clusters are real.**
If they disagree, the clustering is noise.

### Implementation
- Use plotly for interactive 3D rotation
- Generate static PNG for documentation
- Animate over time to see regime transitions

## Milestone 7: Phase Detection (Accumulation / Uptrend / Distribution)

### Expected Feature Signatures

#### Accumulation (Smart Money Buying Before Price Rise)
| Feature | Expected | Why |
|---------|----------|-----|
| Entropy | Low, declining | Market ordered as floor establishes |
| Whale flow | Positive, growing | Smart money buying |
| Volume | Below average | Quiet, no retail attention |
| Spread | Tight | Patient institutional flow |
| Imbalance | Slightly bid-heavy | Persistent buying at bid |
| VPIN | Low | No adverse selection |
| Absorption | High | Volume absorbed without price movement |
| Divergence | Negative | Volume up, price flat |
| Range position | Low (near bottom) | Buying at support |
| Hurst | ~0.5 trending up | Random walk -> persistence |
| Concentration | Rising | Fewer accounts, larger positions |

#### Uptrend
| Feature | Expected | Why |
|---------|----------|-----|
| Entropy | Low, stable | Ordered, predictable trend |
| Whale flow | Positive, aligned with momentum | Riding trend |
| Volume | Rising | Retail joins |
| Momentum | Strong positive | Obvious |
| Hurst | > 0.5 | Persistent |
| R-squared | High | Linear price movement |
| Liq asymmetry | Short-heavy | Shorts getting squeezed |

#### Distribution (Smart Money Selling Into Strength)
| Feature | Expected | Why |
|---------|----------|-----|
| Entropy | Rising | Two-sided flow creates disorder |
| Whale flow | Negative, growing | Smart money selling to retail |
| Volume | High | Both sides active — churning |
| Spread | Widening | Market makers sense toxicity |
| VPIN | High | Informed sellers, uninformed buyers |
| Absorption | High (sell side) | Volume absorbed on sells |
| Divergence | Positive | Volume up, price flat/slightly up |
| Churn | High | Position turnover |
| Range position | High (near top) | Selling into resistance |
| Concentration | Declining | Distributing to many small holders |

### Key Insight
Accumulation and distribution look **identical in price** (range-bound) but
**opposite in flow features** (whale buying vs selling, VPIN low vs high,
divergence negative vs positive). This is where the feature set has theoretical
edge over price-only analysis.

### Multi-Pair Uptrend Detection
```
For each pair:
    regime_score = f(entropy, momentum, whale_flow, absorption, hurst)
    classify: ACCUMULATION | UPTREND | DISTRIBUTION | DOWNTREND | RANGING

Across pairs:
    breadth = count(UPTREND) / N_pairs
    flow_alignment = correlation(whale_flow across pairs)

    if breadth > 0.6 AND flow_alignment > 0.5:
        -> MARKET-WIDE UPTREND
    if breadth > 0.6 AND whale_flow turning negative across leaders:
        -> DISTRIBUTION WARNING (smart money exiting while retail buys)
```
