```
      ⚠️  Inconclusive
      ...

FINAL DECISION:
🔄 PIVOT or 🛑 NO-GO (likely at 2 weeks)
```

### Interpret Checkpoint Results

**✅ Good signs (continue collecting):**
- All features are present (no "Missing column" errors)
- No excessive NaN values
- Some combinations show correlation > 0.03 (even if p > 0.001)
- Data appears clean and reasonable

**⚠️ Warning signs (investigate):**
- Many "Missing column" errors → Check schema mismatch
- Excessive NaN values (>50%) → Check computation logic
- All correlations near zero → May need more data or parameter tuning
- Wildly unrealistic values → Check data source

**❌ Stop and fix:**
- Process crashed / no data after day 3
- All features are zero or NaN
- Parquet files corrupted

### Checkpoint Decision

- **If data looks good**: Continue collecting, wait for Week 4
- **If data has issues**: Fix them now before accumulating more bad data
- **If early signals positive**: Good sign, but still wait for full validation

### Decision Framework

#### 🚀 GO (4-5 Hypotheses Pass)

**Meaning**: Strong statistical evidence for alpha generation.

**Next Steps:**

1. **Paper Trading (Week 5-6)**

2. **Risk Management**
   - Start with small position sizes (1-5% of capital per trade)
   - Set strict stop-losses
   - Monitor daily P&L vs. hypothesis predictions

3. **Live Deployment (Week 7+)**
   - Begin with minimal capital ($100-$1000)
   - Scale up only after 2+ weeks of profitable paper trading
   - Continuously monitor hypothesis validity

**Confidence Level**: HIGH
- Expected Sharpe: 0.5-2.0 (from H5)
- Expected hit rate: 55-65%

---

#### 🔄 PIVOT (2-3 Hypotheses Pass)

**Meaning**: Selective evidence. Some signals work, others don't.

**Next Steps:**

1. **Identify Strong Signals**
   - If H1 passed: Use whale flow as primary signal
   - If H5 passed: Use persistence indicator for regime filtering
   - Ignore hypotheses that failed

2. **Selective Strategy**
   ```
   Example: If only H1 and H5 pass
   → Only trade when:
      - Whale flow is strong (H1)
      - AND persistence indicator > threshold (H5)
   → Ignore entropy, liquidation, concentration signals
   ```

3. **Extended Validation**
   - Collect 4 more weeks of data
   - Re-test the weak hypotheses
   - Maybe they need more data or different parameters

**Confidence Level**: MEDIUM
- Expected Sharpe: 0.3-0.8
- Expected hit rate: 52-58%
- Higher risk of overfitting

---

#### 🛑 NO-GO (0-1 Hypotheses Pass)

**Meaning**: No statistical evidence for alpha. Do NOT deploy real capital.

**Next Steps:**

1. **Extended Data Collection**
   - Continue for 8-12 weeks total
   - Markets may need more time to show patterns
   - Crypto volatility may require longer sampling period

2. **Hypothesis Refinement**
   - Review which tests were closest to passing
   - Adjust thresholds (but beware overfitting!)
   - Consider different time windows

3. **Alternative Approaches**
   - Try different assets (add more coins)
   - Try different timeframes (5min instead of 1min)
   - Add external data (funding rates, perp-spot basis)

4. **Accept Reality**
   - If after 12 weeks still NO-GO → The edge may not exist
   - Don't force it with parameter tuning (overfitting trap!)
   - Better to find a new strategy than lose money on a bad one

**Confidence Level**: LOW / NONE
- Do not trade this strategy
- It's just noise, not signal

## Troubleshooting

### Process Keeps Crashing

**Symptom**: `make run` exits after a few hours.

**Common causes:**
1. **Network issues**: WebSocket disconnects
   - **Fix**: Add auto-reconnect logic (already in code)
   - Check logs: `grep -i "error\|disconnect" logs/*.log`

2. **Memory leak**: Process using >8GB RAM
   - **Fix**: Reduce buffer sizes in config
   - Monitor: `top` or `htop`

3. **Disk full**: No space for new Parquet files
   - **Fix**: Add disk space or reduce retention
   - Check: `df -h`

### No Data Being Written

**Symptom**: No Parquet files in `data/features/`.

**Checks:**
```bash
# 1. Is process running?
ps aux | grep ing

# 2. Check logs for errors
tail -100 logs/ing.log

# 3. Is WebSocket connected?
# Look for "Connected to Hyperliquid" in logs

# 4. Check permissions
ls -la data/features/
```

### Features Are All NaN

**Symptom**: `make show_features` shows NaN for many features.

**Cause**: Insufficient market data to compute features.

**Solution**: Wait 5-10 minutes for buffers to fill. Some features need:
- 60 trades (for 1-min rolling windows)
- 100+ order book updates
- First hour may have incomplete features

### Hypothesis Tests Fail to Load Data

**Symptom**: `make test_hypotheses` shows "Missing column" errors.

**Cause**: Schema mismatch between expected and actual column names.

**Fix**:
```bash
# Check actual schema
parquet-tools schema data/features/*.parquet | head -50

# Compare with expected columns in data_loader.rs
# Update data_loader.rs fallback names if needed
```

## Appendix A: systemd Service File

Save this to `/etc/systemd/system/nat-ingestor.service`:

```ini
[Unit]
Description=NAT Hyperliquid Ingestor
After=network.target

[Service]
Type=simple
User=onat
WorkingDirectory=/home/onat/nat/rust/ing
ExecStart=/home/onat/nat/rust/target/release/ing --config /home/onat/nat/config/ing.toml
Restart=always
RestartSec=10
StandardOutput=append:/home/onat/nat/logs/ing.log
StandardError=append:/home/onat/nat/logs/ing_error.log

# Resource limits (optional)
MemoryLimit=2G
CPUQuota=50%

[Install]
WantedBy=multi-user.target
```

Then:
```bash
sudo systemctl daemon-reload
sudo systemctl enable nat-ingestor
sudo systemctl start nat-ingestor
```
