## 1. Context / why

NAT's binding constraint: **a mid-price edge is not a tradeable edge.** The canonical
case is `imbalance_qty_l1` IC≈0.45 at 1–5 s collapsing to ~0.03 conditional on
realistic fills (taker RT ≈ 11 bps against a 0.5–2 bps move is economically
impossible). Every claimed "winner" must survive an adversarial refutation pass
*before* it earns trust, capital, or the Q1 data investment.

Today this exists only as the LLM **`alpha-skeptic` agent**
(`.claude/agents/alpha-skeptic.md`) — a valuable but one-off, non-reproducible
qualitative review. **Q4 operationalizes that checklist into a reproducible,
re-runnable quantitative gate** that emits a per-algo refutation report and a verdict
expressed against the *imported* gates, with the LLM agent as the qualitative
synthesis layer on top.

## 2. Winners in scope

The deployables from the gauntlet/oos (`reports/gauntlet_*.json`,
`scripts/alpha/oos_window.py`) and the lifecycle (`scripts/signal_lifecycle.py`,
state VALIDATED): **`3f_liquidity`, `jump_detector`, `optimal_entry`,
`funding_reversion`, `surprise_signal`** — per symbol (BTC/ETH/SOL).

## 3. Objective / deliverable

A reproducible gate: `nat skeptic run [--algo X] [--symbol S] [--json]` backed by a new
orchestrator `scripts/exploration/alpha_skeptic_gate.py` that, per (winner, symbol):

1. **Reproduces** the claimed edge from the real tooling (or flags non-reproduction —
   itself a finding).
2. Runs the **failure-mode battery** (§5) as automated quantitative tests, each
   reporting `refuted | survives | untestable_now` + evidence (numbers, file:line,
   command output).
3. Emits `reports/skeptic/<algo>_<symbol>_refutation.json` + a console summary table.
4. Produces an overall verdict — **SURVIVES / KILLED / NOT-PROVEN** — expressed against
   the imported gates (§6), **never invented thresholds**.
5. Records the verdict to the signal lifecycle, provenance-stamped (`git_sha`): a
   KILLED winner → `REJECTED` (or held), blocking promotion; a survivor's VALIDATED
   record is annotated with the refutation provenance.

## 4. The lethal check to ADD (check 0) — fill-conditional / conditional-IC

This is the one the existing tooling does not cover and the one most likely to kill a
micro edge. **Check it first.**

- Re-score each winner's edge under a **realistic trade-flow fill model**, not
  mid-price: use `scripts/kalman/fill_sim.py` (+ `scripts/kalman/limit_order_sim.py`)
  against the raw trade stream in `data/trades/<date>/*.parquet` (present: 2026-06-10..)
  to simulate executable fills (queue position, taker crossing).
- Compute the edge's IC **conditional on achievable fills** → `conditional_ic`.
  **Bar: conditional-IC > 0.15** (imported from the agent spec / the D1 go/no-go).
- Costs strictly via `load_costs()` (`scripts/costs.py` → `config/costs.toml`); report
  which fee tier was used (1.61 bps VIP9 vs 11 bps taker RT changes the verdict).
- If the fill model cannot be applied to a given algo, the verdict is **NOT-PROVEN by
  default** — do not pass on mid-price numbers.
- **Dependency:** verify `fill_sim.py` is wired to `data/trades/`. If it is incomplete,
  finishing a minimal trade-flow fill model is the critical sub-task (this *is* the D1
  conditional-IC go/no-go) and should be split out if it grows.

## 6. Imported gates & thresholds — NEVER invent

- **conditional-IC > 0.15** (fill model).
- **G1:** BH-FDR screen. **G4:** walk-forward + deflated Sharpe — OOS Sharpe > 0.5,
  OOS/IS > 0.7, deflated DSR ≥ 0.95, maxDD < 5 %, ≥ 30 trades, PF > 1.2.
  **G8:** 14-day paper, 5 criteria.
- **Data sufficiency:** ≥ 7 clean days, ≥ 4 folds @ ≥ 500 bars.

## 7. How to implement (files)

- **Reuse / refactor:** `scripts/exploration/skeptical_regression_test.py` — split its
  T1–T10 into reusable functions taking `(signal, fwd_returns, costs, ...)`.
- **New:** `scripts/exploration/alpha_skeptic_gate.py` — per-winner orchestrator: load
  the algo signal via the registry (`scripts/algorithms/`), the forward returns, run
  check 0 (fill-conditional via `fill_sim.py`) + checks 1–10, assemble report + verdict.
- **CLI:** add `nat skeptic run [--algo --symbol --json]`, mirroring the existing

- **Lifecycle:** `scripts/signal_lifecycle.py` — KILLED → `REJECTED`/hold; SURVIVES →

- **LLM layer:** invoke the `alpha-skeptic` agent per winner — it reads the harness JSON
  + code/data, ranks lethality, and writes the GO / NOT-PROVEN narrative against the
  gates. (Harness = numbers; agent = judgement.)

## 8. Verification (methodology — non-negotiable)

1. **Planted-flaw test FIRST.** Construct synthetic "winners" with KNOWN flaws and
   assert the gate KILLS each (the correct check fires): (a) a look-ahead-leaking
   signal, (b) a mid-price-only edge that dies on fills, (c) an overlapping-window-
   inflated IC. A gate that cannot kill a known-bad signal is worthless.
2. **Real-parquet smoke** on the 5 winners against existing `data/features/` +
   `data/trades/`: reports generate, conditional-IC computes via `fill_sim.py`,
   verdicts are gate-expressed.
3. **Known-good vs known-bad reference:** `3f_liquidity` (strongest; expected to
   survive) vs raw `imbalance_qty_l1` (expected to die on fills).

## 9. Sequencing & dependencies

- **Gates Q1.** Run Q4 on existing data *before* the Hetzner box / 90-day streak. If the

- The **data-sufficiency** check (§5.7) will report FAIL today (Q0 verdict: 0 clean

  cost, overlap, regime) are still decisive on existing data.
- Fill model (`scripts/kalman/fill_sim.py`) wiring to `data/trades/` is the one real
  build risk — see §4.
