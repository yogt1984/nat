## Phase 3: Implement Data Loader

### Task 3.1: Create Parquet Reader Module
**Priority:** CRITICAL
**Time:** 2-3 hours

**File:** `rust/ing/src/hypothesis/data_loader.rs`

```rust
//! Parquet Data Loader for Hypothesis Testing
//!
//! Loads feature data from Parquet files and converts to hypothesis test format.

use anyhow::{Context, Result};
use arrow::array::{Array, Float64Array, StringArray, TimestampNanosecondArray};
use arrow::record_batch::RecordBatch;
use parquet::arrow::arrow_reader::ParquetRecordBatchReaderBuilder;
use std::fs::File;
use std::path::Path;

use crate::hypothesis::{H1TestData, H2TestData, H3TestData, H4TestData, FeatureRow};

/// Load all Parquet files from a directory
pub fn load_all_data(data_dir: &Path) -> Result<Vec<RecordBatch>> {
    let mut batches = Vec::new();

    for entry in std::fs::read_dir(data_dir)? {
        let entry = entry?;
        let path = entry.path();

        if path.extension().and_then(|s| s.to_str()) == Some("parquet") {
            let file = File::open(&path)?;
            let builder = ParquetRecordBatchReaderBuilder::try_new(file)?;
            let mut reader = builder.build()?;

            while let Some(batch) = reader.next() {
                batches.push(batch?);
            }
        }
    }

    Ok(batches)
}

/// Extract a Float64 column from batches
pub fn extract_f64_column(batches: &[RecordBatch], column_name: &str) -> Result<Vec<f64>> {
    let mut values = Vec::new();

    for batch in batches {
        if let Some(col) = batch.column_by_name(column_name) {
            if let Some(array) = col.as_any().downcast_ref::<Float64Array>() {
                for i in 0..array.len() {
                    if !array.is_null(i) {
                        values.push(array.value(i));
                    }
                }
            }
        }
    }

    Ok(values)
}

/// Extract timestamps
pub fn extract_timestamps(batches: &[RecordBatch]) -> Result<Vec<i64>> {
    let mut values = Vec::new();

    for batch in batches {
        if let Some(col) = batch.column_by_name("timestamp") {
            if let Some(array) = col.as_any().downcast_ref::<TimestampNanosecondArray>() {
                for i in 0..array.len() {
                    if !array.is_null(i) {
                        values.push(array.value(i));
                    }
                }
            }
        }
    }

    Ok(values)
}

/// Load H1 test data
pub fn load_h1_data(batches: &[RecordBatch], flow_window: &str, return_horizon: &str) -> Result<H1TestData> {
    let whale_flows = extract_f64_column(batches, flow_window)?;
    let returns = extract_f64_column(batches, return_horizon)?;
    let timestamps = extract_timestamps(batches)?;

    // Ensure all arrays have same length
    let min_len = whale_flows.len().min(returns.len()).min(timestamps.len());

    Ok(H1TestData {
        whale_flows: whale_flows[..min_len].to_vec(),
        returns: returns[..min_len].to_vec(),
        timestamps: timestamps[..min_len].to_vec(),
    })
}
```

## Troubleshooting Guide

### Issue: "No Parquet files found"
**Solution:**
- Check data collection is running: `make run`
- Verify output directory: `ls data/features/`
- Check config: `cat config/ing.toml`

### Issue: "Column not found in Parquet"
**Solution:**
- Inspect schema: `parquet-tools schema data/features/[file]`
- Update column names in data_loader.rs
- Check feature extraction is enabled

### Issue: "All tests return INCONCLUSIVE"
**Solution:**
- Not enough data (< 2 weeks)
- Data quality issues
- Feature values all zero/constant
- Check data validation

### Issue: "All tests ACCEPT easily"
**Solution:**
- Possible overfitting
- Check Bonferroni correction is enabled
- Review OOS/IS ratios
- Verify walk-forward validation

### Issue: "Binary crashes when loading data"
**Solution:**
- Check file permissions
- Verify Parquet files not corrupted
- Add error handling in data_loader.rs
- Check memory usage
