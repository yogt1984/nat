## Part 0: Feature Vector Deep Dive — What Is Already Implemented

### 0.1 Entropy Vector (24 features)

Three sub-families:

**A) Permutation Entropy (4 features)** — Bandt-Pompe (2002). Extracts ordinal patterns of length `order=3`, counts frequencies, computes Shannon entropy. Normalized to [0, 1].

| Feature | Window | What it measures |
|---------|--------|-----------------|
| `ent_permutation_returns_8` | 8 ticks | Short-term return predictability |
| `ent_permutation_returns_16` | 16 ticks | Medium-term return predictability |
| `ent_permutation_returns_32` | 32 ticks | Longer-term return predictability |
| `ent_permutation_imbalance_16` | 16 ticks | **BROKEN: hardcoded to 0.0** |

Low PE = returns follow repeated patterns (trending). High = random walk. Multi-window captures scale: 8-tick is microstructure noise, 32-tick is meaningful. PE is non-spectral but responds to periodic signals. Think of it as a non-parametric test for "is there structure at this scale?"

**B) Distribution Entropy (6 features, 4 active)**

| Feature | Input | Status |
|---------|-------|--------|
| `ent_spread_dispersion` | Spread history | **BROKEN: 0.0** |
| `ent_volume_dispersion` | Trade sizes (30s, 10 bins) | Working |
| `ent_book_shape` | Order book depths | Working |
| `ent_trade_size_dispersion` | Trade sizes (30s, 5 bins) | Working |
| `ent_rate_of_change_5s` | Entropy history | **BROKEN: 0.0** |
| `ent_zscore_1m` | Entropy history | **BROKEN: 0.0** |

Low book_shape entropy = liquidity wall/spoofing. High = normal market. Low volume_dispersion = institutional blocks. High = mixed retail+institutional.

**C) Tick Entropy (14 features: 7 raw + 7 volume-weighted)** — Classify each trade as up/down/neutral, count frequencies in window, compute Shannon entropy.

| Window | Raw | Vol-weighted |
|--------|-----|-------------|
| 1s | `ent_tick_1s` | `ent_vol_tick_1s` |
| 5s | `ent_tick_5s` | `ent_vol_tick_5s` |
| 10s | `ent_tick_10s` | `ent_vol_tick_10s` |
| 15s | `ent_tick_15s` | `ent_vol_tick_15s` |
| 30s | `ent_tick_30s` | `ent_vol_tick_30s` |
| 1m | `ent_tick_1m` | `ent_vol_tick_1m` |
| 15m | `ent_tick_15m` | `ent_vol_tick_15m` |

Raw treats all trades equally. Volume-weighted weights by trade size. Divergence between raw and volume-weighted = institutional flow disagrees with retail flow.

The 7 windows are a multi-resolution analysis — similar to wavelet decomposition but using entropy instead of amplitude. Ratio `ent_tick_1s / ent_tick_15m` is a crude spectral slope.

**Summary: 4 of 24 features are hardcoded to 0.0. Effective dimension is 20.**

**Missing extensions:** Spectral entropy (entropy of FFT power spectrum), approximate/sample entropy (self-similarity), transfer entropy (information flow between features).

### 0.2 Trend Vector (15 features)

**A) Momentum (6 features)** — OLS regression slope + R-squared at 3 windows.

| Feature | Window | Interpretation |
|---------|--------|---------------|
| `trend_momentum_60` + `r2` | 60 ticks (~6s) | Immediate direction |
| `trend_momentum_300` + `r2` | 300 ticks (~30s) | Short-term trend |
| `trend_momentum_600` + `r2` | 600 ticks (~1m) | Medium-term trend |

High momentum + high R2 = clean trend. High momentum + low R2 = volatile drift. Low momentum + high R2 = flat and quiet.

**B) Monotonicity (3 features)** — Fraction of consecutive ticks in dominant direction [0.5, 1.0]. Non-parametric, cares only about direction consistency. > 0.7 = strong trend.

**C) Hurst Exponent (2 features)** — R/S analysis at 300 and 600 ticks.
- H < 0.5 = mean-reverting (anti-persistent)
- H = 0.5 = random walk
- H > 0.5 = trending (persistent)

**Most spectral feature.** H relates to spectral density: `S(f) ~ f^(-(2H+1))`.

**D) EMA Crossover (4 features)** — EMA(10) - EMA(50), normalized, plus absolute values.

**Missing:** FFT/Welch spectral, wavelet decomposition, autocorrelation at multiple lags.

### 0.3 Illiquidity Vector (12 features)

Four market microstructure measures at two windows (100, 500 trades) + ratios + composite:

| Algorithm | Reference | What it measures |
|-----------|-----------|-----------------|
| Kyle's Lambda | Kyle (1985) | Regression of price change on signed volume |
| Amihud's Lambda | Amihud (2002) | |return| / volume |
| Hasbrouck's Lambda | Hasbrouck (2009) | |price change| vs sqrt(volume) |
| Roll Spread | Roll (1984) | Implied spread from price autocovariance |

`illiq_kyle_ratio = kyle_100/kyle_500` measures whether illiquidity is changing.

**Missing:** Garman-Klass, Yang-Zhang volatility estimators.

### 0.4 Toxicity Vector (10 features)

**VPIN** — Easley, Lopez de Prado, O'Hara (2012). Volume-synchronized probability of informed trading. At windows 10 and 50. High VPIN = informed traders aggressively taking liquidity. Spiked before 2010 Flash Crash.

Also: `toxic_adverse_selection` (permanent impact), `toxic_effective_spread` (trader cost), `toxic_realized_spread` (MM profit), `toxic_flow_imbalance` (net pressure), `toxic_index` (composite).

**Missing:** EMA of imbalance over time (persistent pressure) — candidate for aggregation step.

### 0.6 Volatility Vector (8 features)

| Feature | Algorithm |
|---------|-----------|
| `vol_returns_1m/5m` | Std(returns) |
| `vol_parkinson_5m` | (H-L)^2/(4 ln2), more efficient than close-close |
| `vol_spread_mean/std_1m` | Spread statistics |
| `vol_midprice_std_1m` | Price variability |
| `vol_ratio_short_long` | vol_1m/vol_5m — crude spectral ratio |
| `vol_zscore` | Current vs historical |

`vol_ratio_short_long` > 1 = choppy (high-freq vol). < 1 = smooth (trending).

### 0.8 Whale Flow Vector (12 features) — Hyperliquid-unique

### 0.9 Liquidation Vector (13 features) — Hyperliquid-unique

Risk mapping above/below 1/2/5/10 pct price moves. Asymmetry, intensity, positions at risk, nearest cluster distance. A liquidation heatmap in feature form.

### 0.10 Regime Vector (20 features)

Minute-scale Wyckoff phase detection:

| Sub-feature | Count | What it detects |
|------------|-------|----------------|
| Absorption | 4 | Volume per unit price change (1h/4h/24h + zscore) |
| Divergence | 5 | Deviation from Kyle's lambda prediction + kyle_lambda |
| Churn | 4 | Two-sided volume (institutional rotation) |
| Range Position | 4 | Where price sits in 4h/24h/1w range [0,1] |
| Composite | 3 | accumulation_score, distribution_score, clarity |

### 0.11 Derived Vector (15 features)

Cross-domain interactions:
- `entropy_trend_interaction` = entropy x (1 - monotonicity). High = choppy.
- `trend_strength_60/300` = sign(momentum) x monotonicity x (1 - entropy)
- `regime_type_score` = low entropy + high vol = breakout; high entropy + high vol = chaos
- `illiquidity_trend` = illiquidity x |momentum|. Informed flow during trends.
- `toxicity_regime` = toxicity x entropy. Toxic choppy markets.
- `regime_indicator` = -1 (trending) to +1 (mean-reverting)

**Warning:** Computed from other vectors. Clustering derived + entropy + trend double-counts.

### 0.12 Context Vector (9 features)

Funding rate, funding zscore, open interest, OI change (5m absolute + pct), premium bps, 24h volume, volume ratio, mark-oracle divergence. Market-level conditions.

### 0.13 Raw Vector (10 features)

Midprice, spread, spread bps, microprice, bid/ask depth at 5 and 10 levels, bid/ask order count at 5 levels. **Warning:** raw levels (e.g. BTC price $60K vs $90K) will cluster by time period, not by market state. Normalize or use returns instead.

### 0.14 Flow Vector (12 features)

Trade count/volume at 1s/5s/30s, aggressor ratio at 5s/30s, VWAP at 5s, VWAP deviation, avg trade size 30s, intensity. Execution pattern analysis.

## Part 1: Clustering

### 1.0 Complete Clustering Algorithm Catalog

#### Partition-based

| Algorithm | How it works | Strengths | Weaknesses |
|-----------|-------------|-----------|-----------|
| K-Means | Minimize within-cluster sum of squares | Fast, interpretable centroids | Spherical assumption, needs k, outlier sensitive |
| K-Medoids (PAM) | Uses actual data points as centers | Robust to outliers | Slow O(n^2 k) |
| Mini-Batch K-Means | K-Means on random subsets | Scales to large data | Slightly worse quality |

#### Model-based

| Algorithm | How it works | Strengths | Weaknesses |
|-----------|-------------|-----------|-----------|
| **GMM** | EM, assumes k Gaussian components | Soft assignments, BIC for k, elliptical | Assumes Gaussian, overfits in high dims |
| DPGMM | Bayesian GMM, automatic k | No need to specify k | Slow, prior sensitive |
| HMM | GMM + temporal transitions | Models persistence | Needs temporal ordering, more params |

#### Density-based

| Algorithm | How it works | Strengths | Weaknesses |
|-----------|-------------|-----------|-----------|
| DBSCAN | Dense neighborhoods clustered | Arbitrary shapes, finds noise | eps sensitive, varying density |
| **HDBSCAN** | Hierarchical DBSCAN | No eps, varying density, identifies noise | Slow, min_cluster_size matters |
| OPTICS | Orders by density reachability | Reachability plot diagnostic | Hard to extract flat clusters |

#### Hierarchical

| Algorithm | How it works | Strengths | Weaknesses |
|-----------|-------------|-----------|-----------|
| **Agglomerative (Ward)** | Bottom-up, minimize within-cluster variance | Dendrogram, no k needed | O(n^2) memory |
| Complete/Average linkage | Different merge criteria | Various properties | Less relevant |
| BIRCH | CF-tree for large data | Very fast | Spherical assumption |

#### Graph-based

| Algorithm | How it works | Strengths | Weaknesses |
|-----------|-------------|-----------|-----------|
| Spectral | Cluster eigenvectors of graph Laplacian | Non-convex clusters | O(n^3) |
| Affinity Propagation | Message passing for exemplars | Auto k | Very slow, too many clusters |

#### Recommendation

**Primary: GMM.** Soft assignments, BIC for k, handles elliptical clusters.
**Secondary: HDBSCAN.** Validates GMM's k, finds outliers.
**Diagnostic: Agglomerative Ward.** Dendrogram visualization.
**Skip everything else initially.** Two algorithms that agree > ten that disagree. Diversify vectors, not algorithms.

### 1.1 Cluster Separability Metrics

#### Internal (no ground truth)

| Metric | Range | Good value |
|--------|-------|-----------|
| Silhouette score | [-1, 1] | > 0.25 meaningful, > 0.5 strong |
| Davies-Bouldin index | [0, inf) | < 1.0 good |
| Calinski-Harabasz | [0, inf) | Higher is better |
| BIC (GMM) | (-inf, inf) | Minimum across k |
| Gap statistic | [0, inf) | Largest gap |
| Dunn index | [0, inf) | Higher is better |

#### Stability

| Metric | Method | Good value |
|--------|--------|-----------|
| Bootstrap ARI | Resample 50x, recluster, compare | > 0.6 |
| Temporal ARI | First half vs second half | > 0.5 |
| Perturbation robustness | Add noise, recluster | ARI > 0.7 |

#### Predictive

| Metric | What it tells you |
|--------|-----------------|
| Kruskal-Wallis test | p < 0.05 = clusters differentiate returns |
| Eta-squared | > 0.01 small, > 0.06 medium effect |
| Cluster-conditional Sharpe | Different Sharpes = different regimes |
| Transition matrix entropy | Low = predictable transitions |

#### Multimodality

| Test | What it detects |
|------|----------------|
| Hartigan's dip test | Unimodal vs multimodal |
| Bimodality coefficient | BC > 5/9 suggests bimodality |
| Silverman's bandwidth test | Number of modes |
| GMM BIC k=1 vs k=2 | Direct test: do clusters exist? |

### 1.2 Dimensionality Reduction and Visualization

#### Linear Methods

| Method | Preserves | When to use |
|--------|-----------|------------|
| **PCA** | Maximum variance | Always first — fast, interpretable |
| Factor Analysis | Latent factors with noise model | Different noise levels |
| ICA | Statistical independence | Non-Gaussian latent factors |
| LDA | Class separation (supervised) | After clustering |

PCA on 24d entropy -> likely 80%+ variance in 3-4 components.

#### Nonlinear Methods

| Method | Preserves | Speed | Parameters |
|--------|-----------|-------|-----------|
| **UMAP** | Local + global topology | Fast | n_neighbors=15, min_dist=0.1 |
| **t-SNE** | Local neighborhoods | Slow | perplexity=30 |
| Isomap | Geodesic distances | Moderate | n_neighbors |
| LLE | Local linear structure | Moderate | n_neighbors |
| Kernel PCA | Nonlinear PCA | Moderate | Kernel choice |
| Diffusion Maps | Diffusion distances | Moderate | alpha |
| Trimap | Local + global | Fast | Minimal tuning |
| PaCMAP | Local + global | Fast | Minimal tuning |

UMAP parameters: n_neighbors=15 (increase to 30-50 for more global), min_dist=0.1 (decrease to 0.01 for tighter clusters), metric="euclidean" (try "cosine" if magnitudes vary).

**Critical caveats:** UMAP/t-SNE inter-cluster distances NOT meaningful. Cluster sizes NOT meaningful. Run multiple seeds.
